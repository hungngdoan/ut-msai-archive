## Why build it

I wanted a single, durable place to track a two-year graduate program — and I wanted the act of
maintaining it to be cheap enough that I'd actually keep it up. A custom static site also doubles as a
small, ongoing engineering artifact, which fits a program in AI better than a hosted notebook would.

## Design decisions

- **One file per entry, schema-validated.** Every course, week, reference, invention, and field note is
  a single content file checked against a zod schema. A bad edit fails the build instead of shipping
  silently. This is also what makes it safe to let an AI assistant propose changes — the schema and CI
  are the guardrails.
- **Derived, honest progress.** Rather than a manual percentage I'd forget to update, course progress is
  computed from term start and end dates. It can be wrong about ambition but never about the calendar.
- **No backend, no database, no login.** Free hosting on GitHub Pages, nothing to operate, nothing to
  leak.

## What I'd add next

A simple tag index across collections, an RSS feed for the experiment log, and an automated link checker
in CI so public resource links can't quietly rot over two years.
